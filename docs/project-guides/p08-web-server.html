P8 Web server

ESP32 tự phát Wi-Fi AP, chạy HTTP server, và cho điện thoại bật/tắt onboard LED GPIO2. Breadboard để trắng.

RobotLab-P8HTTPGPIO2 onboard

An toàn

Mặc định P8 không cắm dây ngoài: chỉ ESP32 qua USB. GPIO ESP32 là logic 3.3V; không đưa 5V vào chân GPIO. Nếu thử servo tùy chọn, servo dùng nguồn 5V riêng và GND phải nối chung với ESP32.

Board map

Chi tiết: boards/p08.md. P8 mặc định là board trống: không cắm ESP32 lên MB102, không nối rail, không cắm LED ngoài.

Kết nối duy nhất: USB máy tính → ESP32. Onboard LED nằm trên chính ESP32, không có tọa độ breadboard.

Các bước làm

  1. Rút ESP32 và mọi dây khỏi breadboard; để MB102 trắng.
  2. Cắm ESP32 vào USB máy tính.
  3. main.cpp đang chọn p08_setup / p08_loop.
  4. Chạy pio run -t upload, rồi pio device monitor.
  5. Điện thoại kết nối Wi-Fi RobotLab-P8, mật khẩu robotlab8.
  6. Mở http://192.168.4.1, bấm ON/OFF/TOGGLE và nhìn LED onboard GPIO2.

Atomic trace

  1. Nút web gọi fetch('/api/led?state=toggle').
  2. ESP32 nhận HTTP request trong server.handleClient().
  3. p08HandleLed() gọi p08WriteLed().
  4. digitalWrite(2, HIGH/LOW) đổi mức logic 3.3V trên GPIO2.
  5. Onboard LED sáng/tắt; breadboard không có dòng điện chạy qua.

Hoàn thành khi

  • Điện thoại mở được web page ESP32 tại 192.168.4.1.
  • Bật/tắt được onboard LED GPIO2 từ browser.
  • Bạn hiểu ESP32 đang vừa là firmware device, vừa là Wi-Fi access point, vừa là HTTP server nhỏ.
